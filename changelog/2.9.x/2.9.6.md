# 2.9.6 — 2026-06-18

## Fixed

- **`pubmed_convert_ids` mixed PMCID batches** — `NcbiService.idConvert` now canonicalizes bare-digit PMCIDs to `PMC`+digits before the PMC ID Converter call, so a batch mixing `PMC3531190` and `3531190` resolves like a homogeneous one instead of being rejected wholesale (HTTP 400). The `pmcid` format hint (both `ID_CONVERT_FORMAT_HINTS` and the tool's `ids` `.describe()`) now states bare digits are accepted. ([#73](https://github.com/cyanheads/pubmed-mcp-server/issues/73))
- **`pubmed_europepmc_search` `abstractSnippet`** — normalized to display-ready plain text via a new `toDisplayText` helper: strips structural JATS/HTML tags, decodes HTML entities, and removes soft hyphens (U+00AD) before truncation, so the snippet budget holds text rather than markup and stray soft hyphens don't corrupt token matching. ([#74](https://github.com/cyanheads/pubmed-mcp-server/issues/74))
- **`pubmed_fetch_articles` `grantList`** — `extractGrants` now decodes the residual `&amp;` that NCBI's double-encoding leaves after the XML parser's single decode pass, so `grantId`/`acronym`/`agency`/`country` surface as `CSR&D` rather than `CSR&amp;D`. ([#74](https://github.com/cyanheads/pubmed-mcp-server/issues/74))
- **`pubmed_europepmc_search` silent-rejection error** — the version-only-envelope throw now passes a distinct diagnosis as the message and an actionable next step as `recovery.hint`, instead of the same string for both. Previously the framework rendered byte-identical `Error:`/`Recovery:` blocks. ([#75](https://github.com/cyanheads/pubmed-mcp-server/issues/75))
- **`pubmed_lookup_mesh` `treeNumbers`** — `extractTreeNumbers` now drops `@`-prefixed values, which supplementary concept records (SCRs) surface in the `TreeNum` field as non-navigable mapped-heading pointers (e.g. `@218176`). `treeNumbers` now holds only real tree numbers and is omitted for SCRs; its `.describe()` documents this. ([#76](https://github.com/cyanheads/pubmed-mcp-server/issues/76))
