# 2.7.11 ‚Äî 2026-06-04

## Added

- **`pubmed_fetch_fulltext` config-aware description** ‚Äî the tool description is now composed at startup from the fallback tiers actually enabled (`EUROPEPMC_ENABLED`, `UNPAYWALL_EMAIL`), so `tools/list` advertises only what the deployment can deliver; PMC EFetch is always present, Europe PMC and Unpaywall clauses appear only when their config is set. ([#65](https://github.com/cyanheads/pubmed-mcp-server/issues/65))

## Fixed

- **`pubmed_fetch_fulltext` `dois` branch** ‚Äî DOI input now calls the PMC ID Converter before PMC EFetch, mirroring the `pmids` branch; a DOI whose article is indexed in PMC returns structured JATS (`viaSource: "pmc"`) instead of a false `unavailable`. Previously the `dois` branch skipped PMC EFetch entirely, giving DOI input a narrower resolution path than `pmids` or `pmcids`. ([#64](https://github.com/cyanheads/pubmed-mcp-server/issues/64))
